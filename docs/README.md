## 🤖 COPILOT 開發觀察筆記

### ✨ AI 開發優勢
- **超越預期的能力**: 本來以為 AI 頂多只能處理基本排版功能，沒想到連色系搭配、動畫參數設定、功能規劃與 UX 建議都能同時提供
- **豐富的程式碼生成**: 會生成大量的元件與 CSS 定義項，雖然維護複雜但都有詳細註解
- **版本控制協助**: 能協助完成 Git 初始化、提交和推送到 GitHub 的完整流程

### ⚠️ 開發挑戰與最佳實踐

#### 🔧 功能開發原則
- **先清理後開發**: 如果要新功能來取代舊功能，建議先把舊功能清除乾淨，再弄新功能，免得 AI 會把這兩個混在一起，然後就修不好了 QQ
- **謹慎使用 Clean Code**: 請 AI clean code 的話，它會把所有東西都重寫一次，但會做到組件拆分、模組化效果，大概花了 10 分鐘，但做完之後，很多做好的功能消失了，所以建議邊開發功能邊 clean code，或是一開始就請他按照一定的程式語言規範
- **建議開發順序**: 先做功能，再做面板。要添加新功能的話，隨著專案變大很容易有 bugs
- **內容管理**: 可以透過讀取 README 整理內容並新增到網頁的資料裡

#### 🏗️ 專案結構挑戰
- **效能下降**: 隨著專案規模擴大，AI 運行速度會大幅降低
- **檢查時間**: 每次調整都需要花費大量時間進行驗證檢查
- **適用場景**: 更適合初期簡單版面設計和基礎功能開發

#### 📱 響應式設計限制
- **開發順序**: 必須從最小螢幕開始設計，最後處理 RWD，否則每次修改都會重新處理所有斷點
- **一致性問題**: 雖然能處理單頁面問題，但不同路由間的排版容易不一致，結構缺乏統一性

#### 🎨 設計能力限制
- **動畫能力**: 製作動畫的能力相對較弱
- **美感設計**: 生成具有美感的物件（如樹葉等）的能力有限

#### 🐛 技術債務管理
- **新功能風險**: 開發新功能（如聊天機器人）時容易產生大量 Bug
- **累積債務**: 越到開發後期，累積的技術債務越多
- **功能恢復**: 測試過程中暫時關閉的功能，容易在測試完畢後忘記恢復

#### 💡 最佳實踐建議
- **版本控制**: 開發新功能前務必進行版本控制
- **漸進開發**: 適合用於簡單版面和基礎設計的快速原型開發
- **定期檢查**: 需要建立完善的測試和檢查機制
- **模組化思維**: 從一開始就按照程式語言規範進行開發

## 🚀 快速開始

1. **複製專案**
   ```bash
   git clone https://github.com/oorongtee/personalWebsite.git
   cd personalWebsite
   ```

2. **本地開發**
   ```bash
   # 使用 Python 啟動本地服務器
   python -m http.server 8000
   
   # 或使用 Node.js
   npx serve .
   ```

3. **在瀏覽器中開啟**
   ```
   http://localhost:8000
   ```
