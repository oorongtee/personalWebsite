## 🤖 COPILOT 開發觀察筆記

### ✨ AI 開發優勢
- **超越預期的能力**: 本來以為 AI 頂多只能處理基本排版功能，沒想到連色系搭配、動畫參數設定、功能規劃與 UX 建議都能同時提供
- **豐富的程式碼生成**: 會生成大量的元件與 CSS 定義項，雖然維護複雜但都有詳細註解
- **版本控制協助**: 能協助完成 Git 初始化、提交和推送到 GitHub 的完整流程

### ⚠️ 開發挑戰與最佳實踐

#### 🔧 功能開發原則
- **先清理後開發**: 如果要新功能來取代舊功能，建議先把舊功能清除乾淨，再弄新功能，免得 AI 會把這兩個混在一起，然後就修不好了 QQ
- **謹慎使用 Clean Code**: 請 AI clean code 的話，它會把所有東西都重寫一次，但會做到組件拆分、模組化效果，大概花了 10 分鐘，但做完之後，很多做好的功能消失了，所以建議邊開發功能邊 clean code，或是一開始就請他按照一定的程式語言規範
- **建議開發順序**: 先做功能，再做面板。要添加新功能的話，隨著專案變大很容易有 bugs
- **內容管理**: 可以透過讀取 README 整理內容並新增到網頁的資料裡

#### 🏗️ 專案結構挑戰
- **效能下降**: 隨著專案規模擴大，AI 運行速度會大幅降低
- **檢查時間**: 每次調整都需要花費大量時間進行驗證檢查
- **適用場景**: 更適合初期簡單版面設計和基礎功能開發

#### 📱 響應式設計限制
- **開發順序**: 必須從最小螢幕開始設計，最後處理 RWD，否則每次修改都會重新處理所有斷點
- **一致性問題**: 雖然能處理單頁面問題，但不同路由間的排版容易不一致，結構缺乏統一性

#### 🎨 設計能力限制
- **動畫能力**: 製作動畫的能力相對較弱
- **美感設計**: 生成具有美感的物件（如樹葉等）的能力有限

#### 🐛 技術債務管理
- **新功能風險**: 開發新功能（如聊天機器人）時容易產生大量 Bug
- **累積債務**: 越到開發後期，累積的技術債務越多
- **功能恢復**: 測試過程中暫時關閉的功能，容易在測試完畢後忘記恢復

#### 💡 最佳實踐建議
- **版本控制**: 開發新功能前務必進行版本控制
- **漸進開發**: 適合用於簡單版面和基礎設計的快速原型開發
- **定期檢查**: 需要建立完善的測試和檢查機制
- **模組化思維**: 從一開始就按照程式語言規範進行開發

## 🚀 快速開始

1. **複製專案**
   ```bash
   git clone https://github.com/oorongtee/personalWebsite.git
   cd personalWebsite
   ```

2. **本地開發**
   ```bash
   # 使用 Python 啟動本地服務器
   python -m http.server 8000
   
   # 或使用 Node.js
   npx serve .
   ```

3. **在瀏覽器中開啟**
   ```
   http://localhost:8000
   ```

### 🎯 設計系統核心原則

#### 1. **視覺舒適度 (Visual Comfort)**
- **放寬行距**: 1.6-1.75em 的寬鬆行距，提升長文本閱讀體驗
- **暖色調陰影**: 咖啡色系陰影 `rgba(139, 69, 19, 0.1-0.3)` 創造溫暖感
- **柔和圓角**: 12-16px 統一圓角，避免過於銳利的視覺感受
- **毛玻璃質感**: `backdrop-filter: blur(10px)` 現代玻璃效果

#### 2. **微互動哲學 (Micro-Interaction Philosophy)**
- **hover 縮放**: 所有可互動元素 hover 時輕微縮放至 1.02x
- **按鈕回饋**: 點擊時縮放至 0.98x，模擬真實世界的按壓感
- **滾動顯現**: 元素進入視窗時從 12px 下方滑入並淡入
- **流暢過渡**: 統一 0.3s ease-in-out 過渡時間，確保動畫一致性

#### 3. **漸進式揭露 (Progressive Disclosure)**
- 天氣資訊預設隱藏，點擊後展開詳細資訊
- Works 詳情面板按需顯示，避免資訊過載
- 透明玻璃導航避免遮擋內容，保持視覺層次

#### 4. **回饋與確認 (Feedback & Confirmation)**
- 所有互動都有視覺回饋（懸停、點擊效果）
- 表單提交有成功/失敗狀態提示
- 載入狀態有適當的等待提示

#### 5. **設計統一性 (Design Consistency)**
- **跨頁面一致**: 統一的卡片系統、色彩和間距規範
- **視覺層次**: 相同功能在不同頁面保持一致的操作方式
- **響應式和諧**: 各裝置上保持相同的視覺比例和體驗

#### 6. **產品導向內容 (Product-Focused Content)**
- **使用者價值**: 內容從個人視角轉為使用者受益導向
- **專業定位**: 突出 Product Manager 和 Technical Leader 專業形象
- **emoji 標題**: 統一的 emoji + 英文標題系統，增加親和力

#### 7. **可及性與效能 (Accessibility & Performance)**
- 響應式設計確保各尺寸螢幕可用性
- 高對比度的大地色調確保可讀性
- CSS 動畫使用 transform 和 opacity 提升效能
- 鍵盤導航和觸控友善設計
